# epcsaft 1.5.1

`epcsaft` 1.5.1 is a package-usability release focused on easier installation, editable source workflows, and cleaner user documentation.

## Install

Once PyPI publishing is enabled:

```powershell
python -m pip install epcsaft
```

Current GitHub release/tag install:

```powershell
python -m pip install "epcsaft @ git+https://github.com/tannerpolley/ePC-SAFT.git@v1.5.1"
```

With `uv`:

```powershell
uv add "epcsaft @ git+https://github.com/tannerpolley/ePC-SAFT.git@v1.5.1"
```

Editable local checkout:

```powershell
git clone https://github.com/tannerpolley/ePC-SAFT.git
cd ePC-SAFT
python -m pip install -e .
```

Source and editable installs require Python `>=3.9`, a C++ compiler, CMake, and Ninja or another working CMake generator. Python 3.13 is the current smoke-test baseline.

## Changes

- Added PEP 660 editable-install hooks to the custom build backend.
- Kept the backend's isolated native build directory behavior for editable installs.
- Added tests for editable-hook routing.
- Updated README and Sphinx install pages with pip, GitHub tag, local source, editable, and optional IPOPT install paths.
- Reorganized the docs index so new users see install and usage pages first, while maintainer workflows are still available under source/contributor sections.
- Removed stale user-doc language that framed pages as coding-agent-only workflows.
