# epcsaft 1.5.0

`epcsaft` 1.5.0 is the first official source-release baseline for the native-backed Python package.

## Install

Install from the release tag:

```powershell
python -m pip install "epcsaft @ git+https://github.com/tannerpolley/ePC-SAFT.git@v1.5.0"
```

With `uv`:

```powershell
uv add "epcsaft @ git+https://github.com/tannerpolley/ePC-SAFT.git@v1.5.0"
```

If a platform wheel is attached to this GitHub release, users can install that wheel directly:

```powershell
python -m pip install C:\path\to\epcsaft-1.5.0-*.whl
```

Source builds require Python `>=3.9`, a C++ compiler, CMake, and Ninja or another working CMake generator. Python 3.13 is the current smoke-test baseline.

## Highlights

- Native C++ ePC-SAFT runtime with a Python user API.
- Mixture/state construction through `ePCSAFTMixture` and `ePCSAFTState`.
- Pressure closure, direct-density closure, `rho_guess` continuation, and density-consistency diagnostics.
- Fugacity coefficients, activity coefficients, residual properties, and runtime build/capability diagnostics.
- User-owned parameter-folder workflow through `create_parameter_template(...)` and `ePCSAFTMixture.from_dataset(...)`.
- Native-backed neutral phase equilibrium, electrolyte LLE, fixed-liquid electrolyte bubble pressure, reactive speciation, sequential reactive equilibrium, and scoped reactive electrolyte bubble workflows.
- Optional IPOPT residual-minimization refinement through `epcsaft[ipopt]` when platform prerequisites are available.

## Notes

- The package metadata is prepared for PyPI under the name `epcsaft`, but the current official download path is GitHub release/tag installation.
- Source-checkout reference datasets are useful for validation and examples, but users should own their project parameter folders.
- Full downstream applications should own their own data, balances, run matrices, and acceptance criteria.
